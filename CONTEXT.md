# Wearless Studio

패션 셀러가 상품 사진 몇 장으로 AI 상세페이지를 만들고 편집·다운로드하는 스튜디오. 이 문서는 용어집이다 — 데이터 shape는 `documents/common_data_contract.md`, 제품 요구사항은 `documents/PRD.md`.

## Language

### 작업 단위

**프로젝트 (Project)**:
상세페이지 1건을 만드는 작업 단위. 제품 입력부터 에디터 산출물까지의 전 과정과 결과를 묶는 최상위 개념.
_Avoid_: detailPage(엔티티명으로 금지), 작업, 세션

**상세페이지**:
프로젝트의 산출물. 에디터 블록을 세로로 이어 붙인 최종 이미지 문서.
_Avoid_: 디테일 페이지, 랜딩

**보관함 (Library)**:
프로젝트 목록. 카드를 열면 해당 프로젝트의 에디터로 들어간다.

**크레딧**:
생성 행동의 과금 단위. 소모 전에 예상량을 반드시 예고한다.

### 상품과 생성예시

**의류 종류 (ClothingType)**:
판매하는 상품이 무엇인지 나타내는 분류. 상의·하의·아우터·원피스를 쓰며, 셔츠는 판매 분류에 따라 상의 또는 아우터가 될 수 있다. 열어 입은 모습만으로 의류 종류가 바뀌지는 않는다.
_Avoid_: 상품 카테고리, 생성예시 범위

**생성예시 적용 범위**:
하나의 생성예시를 어떤 의류 종류의 상품에서 보여주고 사용할 수 있는지 나타내는 범위. 예시 원본의 의류 종류와 별개이며, `상의·아우터 공용`은 두 상품 모두에서 같은 구도·분위기를 참고할 수 있다는 뜻이다.
_Avoid_: 생성예시 카테고리, 상체 카테고리

**생성예시 참고 범위 (RefScope)**:
선택한 생성예시에서 무엇을 참고할지 나타내는 값. 전부(`all`)·포즈만(`pose`)·배경만(`bg`)을 쓰며, 적용 가능한 의류 종류와는 다른 축이다. 포즈만은 자세만 참고하고 풀샷·중간샷 프레이밍은 현재 카드가 정한다. 포즈·배경 전용 자산이 없으면 전부 이미지로 대신하지 않는다(ADR-0009).
_Avoid_: 생성예시 적용 범위, 포즈와 프레이밍 함께 복사

### 컷과 블록

**콘티 섹션 (StoryboardSectionRole)**:
상세페이지에서 사진이 설명하는 큰 순서. 핵심 장점(`benefit`) → 핏·코디(`fit`) → 제품 확인(`product`)의 세 섹션을 쓴다. 구매 정보는 콘티 섹션이 아니라 에디터에서 넣는다 (ADR-0005).

**사진 목적 (ContentRole)**:
콘티 카드가 사진으로 무엇을 설명할지 나타내는 내부값. 사용자가 역할명을 직접 고르지 않으며, 핵심 장점은 카드 순서, 핏·코디는 선택한 컷 종류, 제품 확인은 선택한 샷으로 정한다(ADR-0005).

**컷 종류 (CutType)**:
AI가 이미지를 만드는 촬영 방식이자 생성 레시피. 스타일링·호리존·제품·거울 넷이며, 콘티 인스펙터에서 섹션별 허용 컷을 고르지만 상세페이지 섹션 이름으로 쓰지는 않는다(ADR-0003~0005).
_Avoid_: 콘티 섹션, 사진 목적

**스타일링컷**:
모델이 매칭 의류와 함께 일상 맥락에서 착용한 이미지를 만드는 내부 레시피.
_Avoid_: 일상컷, daily

**호리존컷**:
무지 호리존 배경에서 핏과 실루엣을 보여주는 이미지를 만드는 내부 레시피.
_Avoid_: 스튜디오컷, studio

**제품컷**:
모델 없이 제품만 보여주는 내부 생성 레시피. 서비스 샷은 고스트·디테일 둘만 쓴다. 플랫레이는 별도 샷이 아니라 고스트 생성예시의 표현 방식이며, 디테일은 입력한 사진에서 실제로 확인되는 부분만 만든다(ADR-0008).

**아우터 열림 정도**:
아우터 착용컷에서 앞부분을 얼마나 열어 보여줄지 정하는 조건. 전체 열림·부분 열림·전체 닫힘 셋 중 하나다.
_Avoid_: 여밈 상태, 착용 상태, 입는 방식

**중간샷**:
서비스 착용컷의 머리~허리(상의·아우터·원피스) 또는 다리~허리(하의) 프레이밍. 저장값은 `medium`이다. 생성예시 선별에서는 수집 원본의 무릎샷과 미디움샷을 함께 비교하며 선별 전용 토큰 `medium_knee`를 쓰지만, 서비스에는 이 토큰을 쓰지 않는다 (ADR-0007).
_Avoid_: 선택판에서 무릎샷과 미디움샷을 별도 묶음으로 나누기

**거울샷**:
거울 앞 셀피 느낌의 착용 이미지를 만드는 내부 레시피. 방향 개념이 없고 샷은 풀샷·중간샷만, 얼굴 기본값은 '폰으로 가림', 포즈는 셀피 구도로 자동 고정. 거울 장소는 설정이 아니라 생성예시가 담는다 (ADR-0004, ADR-0007).
_Avoid_: 셀피컷, OOTD컷

**내 이미지**:
사용자가 직접 업로드해 그대로 쓰는 이미지. 출처는 `mine`, 현재 사진 목적은 `custom`으로 저장하며 어느 큰 섹션에 놓였는지는 별도로 정한다.

**블록**:
상세페이지를 구성하는 세로 적층 단위. 콘티보드 블록(생성 계획)과 에디터 블록(편집 대상)이 있다.

**예전 블록 종류 (Legacy BlockKind)**:
전환 전에 쓰던 후킹·스타일링·호리존 등의 옛 저장 분류. 테스트 프로젝트를 리셋했으므로 새 콘티에서 읽거나 변환하지 않는다. 현재 페이지 분류는 세 콘티 섹션이고, 컷 종류는 카드의 촬영 방식이며 사진 목적은 내부 자동값이다(ADR-0005).

**자동 블록**:
생성 완료 시 자동으로 덧붙는 안내 블록 — 사이즈 안내, 세탁 안내, AI 생성 안내.

**현재 이미지 수정 (Vary)**:
선택한 이미지를 기준으로 방향·샷·포즈·표정·배경을 바꾼 새 이미지를 만드는 에디터 기능. 원본은 유지하며, 변경 0개로 실행하면 '비슷한 컷 만들기'다.
_Avoid_: 컷 변형

### 생성 플로우

**분석 (Analysis)**:
AI가 제안하고 사용자가 확인·수정하는 상품 속성 묶음(세부 카테고리·성별·핏·소재·강조 특징·모델·매칭 의류).

**실측**:
사용자가 직접 입력하는 cm 단위 치수. AI는 절대 추정하지 않는다. 상품 정보(Product)의 일부다.

**강조 특징 (Selling Point)**:
카피와 이미지 생성 방향에 쓰이는 핵심 셀링 포인트. 최대 5개.

**마네킹컷**:
실제 이미지 생성 전 핏·실루엣 기준을 잡는 마네킹 착용 이미지. 한 번에 한 컷을 만들고 재생성 결과는 버전 히스토리로 남긴다.

**조정 (Adjust)**:
선택한 마네킹컷의 총기장·핏을 바꾸는 행동. 횟수 제한은 없으며, 변경을 실제 이미지에 반영할 때 재생성 크레딧을 쓴다.

**매칭 의류**:
모든 착용 이미지에서 판매 상품과 함께 입힐 사용자 선택 의류. 제품 단독컷에는 적용하지 않는다. 첫 번째 선택이 메인, 두 번째가 서브.

**콘티보드**:
생성 전에 컷 구성·순서·조건을 정하는 계획 보드. 썸네일은 방향을 잡기 위한 예시이며 최종 이미지가 아니다.
_Avoid_: 초안 페이지

**사진 양 (ComposeMode)**:
같은 섹션 순서를 유지하면서 사진 수만 정하는 선택값. 기본형(`basic`)과 확장형(`extended`)만 쓴다.
_Avoid_: 간단형, 서로 다른 페이지 구조

**모델**:
AI 생성 착용컷에 등장하는 가상 인물 프리셋.

### 색상과 이미지

**기준 색상**:
의류 구조와 디테일 분석의 대표 색상 그룹(색상 1). 앞면 이미지가 필수다.
_Avoid_: 베이스 컬러

**추가 색상**:
색상 옵션 표현용 색상 그룹. 최대 3개, 정면 사진 필수.

**스와치 (Swatch)**:
12색 고정 팔레트의 색상 식별자. 색상 그룹의 정체성은 스와치로 정해지며, 고르기 전에는 '색상 미정'이다.

**모노톤**:
스와치 중 화이트·그레이·블랙·아이보리·베이지.

**의류 탭 (Wardrobe)**:
에디터에서 캔버스에 넣을 수 있는, 색상 그룹별로 묶인 이미지 라이브러리.
_Avoid_: 갤러리
