# 생성예시 선별에서는 무릎샷과 미디움샷을 중간샷으로 합친다

상태: Accepted
결정일: 2026-07-18

생성예시 대표 이미지를 고르는 선택판에서는 착용컷의 `knee`와 `medium` 원본을 `medium_knee` 한 묶음으로 합치고 화면에는 `중간샷`으로 표시한다. 스타일링·호리존·거울샷, 남녀와 상의·하의·아우터에 같은 원칙을 적용한다. 다만 성별과 의류 종류는 계속 나누므로 상의 중간샷과 하의 중간샷을 한 묶음에 섞지 않는다.

수집 원본 `CATALOG.json`, 이미지 경로, 후보의 `sourceShot`은 다시 쓰지 않는다. 원본이 무릎으로 기록됐는지 미디움으로 기록됐는지는 증거로 보존하고, 선택 목적지와 선택판 JSON에서만 `medium_knee`를 정본으로 쓴다.

기존 선택 결과를 옮길 때 두 묶음에서 고른 사진의 합이 6장을 넘으면 사진을 자동 삭제하지 않는다. 전부 선택된 상태로 보존하고 해당 묶음을 미완료로 표시해, 사용자가 6장 이하로 줄인 뒤 완료하도록 한다.

## 이유

- 실제 수집 사진에서 무릎과 미디움의 경계가 일관되지 않아 같은 구도를 서로 다른 이름으로 나누는 일이 반복됐다.
- 생성예시의 목적은 정확한 촬영 거리 값을 입력받는 것이 아니라 사용자가 참고할 대표 구도를 고르는 것이므로, 경계가 겹치는 두 목록을 따로 유지할 실익이 작다.
- 원본 표시는 보존하므로 이후 모델 실험에서 두 프레이밍을 다시 나눌 필요가 생겨도 근거를 잃지 않는다.

## 2026-07-18 후속 결정 — 서비스 착용컷도 두 단계로 통합

오너 결정으로 서비스의 착용컷 샷도 `full | medium` 두 값만 쓴다. 화면 라벨은 `풀샷 / 중간샷`이며, 중간샷 저장값은 기존 `medium`을 재사용한다. 서비스의 `knee`와 `close`는 폐지하고, 유입되면 해당 사진 목적의 기본 샷으로 방어 정규화한다. 제품 단독컷은 이후 ADR-0008에서 `ghost | detail` 두 값으로 통합했다.

`medium_knee`는 계속 생성예시 수집·선별 전용 토큰이다. 수집 원본의 `sourceShot`과 파일 경로에 남은 `knee`·`medium`도 연구 증거이므로 다시 쓰지 않는다.

## 최초 결정 당시 범위 밖

최초 결정 시점에는 생성예시 수집·선별 데이터에만 적용하고 서비스 옵션 통합을 별도 결정으로 남겼다. 위 후속 결정으로 그 별도 결정이 완료됐다.
