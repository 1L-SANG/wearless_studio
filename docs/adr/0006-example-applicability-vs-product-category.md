# 생성예시의 원본 의류 종류와 적용 의류 종류를 분리한다

상태: Accepted
결정일: 2026-07-18

상품의 `Product.clothingType`은 판매 상품의 정체성으로 `top | bottom | outer | dress`를 그대로 유지한다. 셔츠를 열어 입었다는 이유로 상의가 아우터가 되지는 않으며, 셔츠는 판매 분류에 따라 `top + shirt` 또는 `outer + shirt`가 된다.

생성예시는 원본 사진 속 상품 종류 `sourceClothingType`과, 예시를 실제로 사용할 수 있는 목표 상품 종류 `applicableClothingTypes`를 따로 기록한다. 목표 `Product.clothingType`이 적용 목록에 있을 때만 예시를 보여주고 사용한다. `[top, outer]`는 화면에서 `상의·아우터 공용`으로 표시하되 `upper` 같은 새 의류 종류는 만들지 않는다.

엄격하게 종류별로 전부 분리하면 풀샷의 공통 구도·포즈·배경을 재사용하지 못하고 아우터 표본 부족이 커진다. 반대로 상의와 아우터를 완전히 합치면 제품 형태·레이어링·여밈처럼 종류에 민감한 정보를 흐린다. 따라서 착용 풀샷처럼 구도와 분위기가 중심인 예시는 사람 검토를 거쳐 공용으로 둘 수 있고, 제품 전체·디테일과 종류별 형태가 중요한 가까운 착용샷은 기본적으로 원본 종류에만 적용한다.

## 영향

- 같은 예시를 종류별로 복제 등록하지 않고 하나의 id와 적용 목록으로 공유한다.
- 공용 후보를 나중에 중간샷(수집 원본 무릎·미디움)·제품·디테일로 다시 분류하면 적용 범위를 원본 종류 하나로 좁힌다. 스타일링·호리존 풀샷일 때만 공용 범위를 유지한다.
- 예시 원본의 옷은 생성할 상품의 정체성이나 구조 근거가 아니다. 상품 입력과 목표 의류 종류가 생성 결과를 결정한다.
- `아우터 열림 정도`는 예시 원본이 아니라 목표 상품이 아우터인지에 따라 적용한다.
- 생성예시 선택 데이터와 서비스 레지스트리는 적용 목록이 비어 있거나 중복되거나 원본 종류를 포함하지 않는 항목을 거부한다.
