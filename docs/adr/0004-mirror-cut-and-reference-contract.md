# 거울샷을 네 번째 컷 종류로 추가하고, 예시·레퍼런스의 기여 규칙을 계약으로 정한다

에이블리 탑셀러 분석(style-room.ai 트렌드 리포트 검토, 2026-06)에서 거울샷(거울 앞 셀피형 리얼 착용컷)이 리뷰 이미지의 문법으로서 강한 설득력을 갖는 것이 확인됐다("리뷰 이미지 1장 = 상세 이미지 10장"). `CutType`에 `mirror`를 추가한다. 거울샷은 **방향(정면/측면/후면) 개념이 없고**(면 커버리지가 무의미), 샷은 `full`/`knee`만 쓰며, 얼굴 기본값은 `hide`(폰으로 가림)다. 거울 장소(피팅룸·방·엘리베이터 등)는 설정 노브가 아니라 **생성예시 이미지가 담는다** — 예시를 고르면 그 장소를 따른다. BlockKind는 불변: 거울샷 블록은 `kind='styling'` + `cutType='mirror'`로, 섹션 역할과 컷 종류의 직교(ADR-0003)를 유지한다. (2026-06-29)

같은 결정에서 분위기 예시·레퍼런스의 기여 계약을 함께 확정한다:

- **생성예시 선택 = "예시 그대로, 옷·모델만 교체."** 예시는 제작 시 메타데이터(샷·각도·조명·배경·포즈)를 태깅하고, 선택 시 그 값이 생성 설정으로 복사되며 이미지 자체도 스타일 레퍼런스로 주입된다.
- **내 레퍼런스(업로드)의 베끼기 규칙**: 항상 반영=조명·색감·공간 무드 / 조건부=포즈 뉘앙스(명시 설정과 충돌 시 무시) / 금지=레퍼런스 속 옷·모델·컷 종류 대분류. 업로드는 모든 컷에서 허용하고(차단 없음, 방어는 이 계층 규칙), `refImages`는 블록 소속이므로 **프로젝트 한정** — 전역 자산으로 저장하지 않는다.
- **방향 토큰은 면(面) 분류다**: `front`는 정면~살짝 비튼 각도까지 포함하는 대역(band). 정면 설정 + 살짝 비튼 예시는 충돌이 아니며 뉘앙스가 통과된다. 뒷면/사이드 설정 + 정면 계열 예시는 분위기(조명·톤)만 반영하고 UI에 한 줄로 안내한다(팝업 금지).
- **공간 무드 유지 컷**: `spaceGroupId`가 같은 블록들은 같은 공간(배경 파라미터)에서 생성한다. `spaceVariation` 기본 `subtle`(같은 구도 미세 이동 — 옷 정확성 우선), 옵션 `varied`(포즈·프레이밍 변주 — 재생성 루프로 받침).

## Consequences

- 토큰·계약 갱신(이번 변경): `lib/types.js`(CutType.MIRROR, SpaceVariation, StoryboardBlock 필드), `mock/db.js` cutTypes, `Storyboard.jsx` MoodGuide 분기(mirror→styling 플레이스홀더), `common_data_contract.md` §3.4·enum 표.
- 계약의 UI 강제는 최소 구현으로 즉시 적용한다(이번 변경): 콘티 인스펙터·에디터 새 컷 패널·변형 패널에서 mirror 선택 시 방향 제거, 샷 full/knee 제한, 얼굴 기본 'hide'(라벨 '폰으로 가림'), 포즈 자동 고정(변형 카테고리에서도 제외). 시각 개편(예시 갤러리 승격, 아이콘 샷 필터, 업로드 타일 통합)은 후속 단계.
- 생성 파이프라인에서 mirror는 스타일링 계열 단계(genSteps `styling`)로 매핑한다.
- 생성예시 수집 카탈로그에 거울샷 버킷과 메타데이터 태깅 스키마 추가가 필요하다(수집 확정 후 별도 논의).
- 무릎샷·미디움·클로즈업의 크롭은 품목에 따른다(상의=상체 위주, 하의=하체 위주) — 예시 버킷 규칙과 동일, 상품 정보에서 자동 결정.
