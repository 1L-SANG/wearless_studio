# 제품컷 샷은 고스트샷과 디테일샷 두 종류로 통합한다

상태: Accepted
결정일: 2026-07-19

서비스에서 사용자가 고르고 저장하는 제품컷 샷은 `ghost | detail` 두 값만 쓴다.

- `ghost`(고스트샷)는 모델 없이 옷 전체를 보여준다. 기본 표현은 사람이 입은 듯한 부피와 실루엣이 있는 고스트 마네킹 방식이다.
- 생성예시가 옷을 펼쳐 위에서 찍은 플랫레이 표현이면, 같은 `ghost` 샷 안에서 그 표현과 프레이밍을 따른다. 플랫레이는 별도 서비스 샷이 아니라 고스트샷의 예시 주도 표현 방식이다.
- `detail`(디테일샷)은 입력 사진에서 실제로 확인되는 부위만 확대한다. 보이지 않는 구조·재질·부자재는 만들지 않는다.

기존 `flatlay` 저장값이 유입되면 `ghost`로 방어 정규화한다. 프로젝트 데이터는 리셋 전제이므로 별도 저장 마이그레이션은 만들지 않는다. 수집 원본과 생성예시 후보의 `sourceShot`·`presentationMethod`에는 `flatlay`를 증거로 보존할 수 있지만, 선택 목적지와 서비스 저장값은 `ghost`다.

기본 콘티의 제품 확인 시드는 다음과 같다.

- 기본형: `ghost(front)` + Detail 입력이 있으면 `detail`, 없으면 `ghost(back)` — 전체 12컷 유지
- 확장형: `ghost(front)` + `ghost(back)` + Detail 입력이 있으면 `detail` — 기존 컷 수 유지

생성예시 선택판에서는 기존 ghost·flatlay 후보와 선택을 하나의 고스트샷 묶음으로 합친다. 원본 표현 방식은 후보별 `presentationMethod`로 보존한다. 합칠 때는 선택을 먼저 모두 복원하고, 최종화 단계에서 일반 묶음 최대 6장 규칙을 적용하되 제외 대상과 이유를 기록한다.

ADR-0007의 착용컷 `full | medium` 통합과 같은 원칙으로 서비스 선택 토큰을 줄이되, 수집 원본 증거는 다시 쓰지 않는다.

## 영향

- `ProductShotType`은 `ghost | detail`이다.
- 제품 전체의 방향은 계속 `front | back`을 쓴다.
- 제품용 행거샷은 계속 제공하지 않는다.
- 프롬프트는 고스트샷의 기본 부피 표현과 플랫레이 생성예시가 있을 때의 예외 표현을 한 섹션에서 설명한다.
