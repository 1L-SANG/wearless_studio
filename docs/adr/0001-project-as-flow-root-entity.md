# 생성 플로우의 최상위 엔티티는 project다

입력→분석→마네킹→콘티→생성→에디터→보관함을 관통하는 식별자가 없어 단계 간 데이터가 mock 싱글톤으로만 이어졌다(`/editor/:id`도 읽히지 않음). 플로우 전체와 산출물을 `Project { id, status: draft|generating|done }` 하나로 묶고, 모든 API가 `projectId`를 첫 인자로 받는다. PRD §13.4의 "프로젝트 id" 용어와 일치한다. (2026-06-11)

## Considered Options

- **detailPage를 엔티티명으로** — 산출물 명칭과 작업 단위가 결합돼, 한 상품으로 여러 산출물을 만드는 P2 확장(다중 상품 일괄 생성) 시 이름이 좁아져 기각. '상세페이지'는 산출물 지칭으로만 쓴다.
- **도입 보류(단일 현재 draft 유지)** — 보관함 재진입·에디터 영속화가 계약 없이 남아 백엔드 시점에 재설계가 필요해져 기각.
