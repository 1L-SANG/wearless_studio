# 생성예시 참고 범위는 전용 자산으로 격리하고 단계적으로 연다

상태: Accepted
결정일: 2026-07-19

## 맥락

하나의 완성 사진을 `전부`, `포즈만`, `배경만`에 공통으로 첨부하면 지시문만으로는 예시의 옷·소품·배경·자세가 서로 새는 문제를 막지 못했다. 특히 포즈만 참고할 때 원본 사진의 크롭까지 따라가면 상의·하의와 풀샷·중간샷 조합이 바뀔 때 잘못된 프레이밍이 전달된다.

따라서 생성예시 하나는 안정된 `exampleId`를 공유하되, 실제로 참고할 픽셀은 범위별 전용 자산으로 분리한다. 생성예시 적용 의류 종류는 ADR-0006의 `applicableClothingTypes`, 참고할 사진 정보는 이 ADR의 `refScope`로 관리하며 두 축을 섞지 않는다.

## 결정

### 1. 세 가지 참고 범위

- `all`(전부): 장소·조명·분위기·포즈·프레이밍·구성을 함께 참고하는 완성 사진이다. 예시 속 옷·신발·액세서리와 모델 정체성은 가져오지 않는다.
- `pose`(포즈만): **자세만** 참고한다. 자산은 옷과 배경이 없는 투명 PNG의 중립 전신 마네킹이다. 좌우 팔다리 위치, 관절 각도, 체중 분배와 몸 방향을 보존하지만 원본 사진의 크롭·화면 배치·카메라 거리와 몸 크기는 복사하지 않는다. 풀샷·중간샷 프레이밍은 현재 콘티 카드의 `shot`과 `cutType`이 정한다.
- `bg`(배경만): 사람·의류·신발·휴대폰·가방과 착용/휴대 소품을 제거한 빈 장소 플레이트다. 장소·조명·분위기만 참고하며 예시의 포즈·몸 방향·프레이밍은 전달하지 않는다. 현재 카드의 생성 조건이 인물과 프레이밍을 정한다.

판매 상품과 사용자가 고른 매칭 의류가 착용컷 의류의 유일한 근거다. 생성예시의 옷은 어느 범위에서도 가져오지 않는다. 제품컷에는 매칭 의류를 적용하지 않는다.

거울 포즈는 얼굴을 가리는 손 위치를 읽을 수 있도록 로고·화면 내용·버튼·케이스 디테일이 없는 단순한 빈 휴대폰 모양 하나만 허용한다.

### 2. 전용 자산이 없으면 안전하게 닫는다

`pose`나 `bg` 전용 파일이 없을 때 `all` 이미지로 대신하지 않는다. 서버는 해당 범위의 이미지 첨부를 생략하고, UI는 실제로 검증·발행된 variant만 선택지로 보여준다. 로컬 manifest에 미래 출력 경로가 적혀 있다는 사실만으로 사용 가능한 자산으로 보지 않는다.

같은 장소 묶음(`spaceGroupId`)은 별도 공간 기준을 사용하므로 생성예시를 `pose`로 강제한다. 따라서 같은 장소 UI에서도 포즈 전용 자산이 실제로 있는 예시만 선택할 수 있어야 한다.

제품 `ghost | detail` 예시는 `all`만 사용하며 `pose`·`bg`를 만들거나 노출하지 않는다(ADR-0008).

### 3. 단계적 공개

착용 생성예시 185개 중 성별·내부 생성 방식·샷·의류 종류를 나눠 고른 같은 19개(10.27%)에 `pose`와 `bg`를 모두 만들어 먼저 검증했다. 10%는 영구 상품 상한이 아니라 전체 확장 전 품질 게이트다.

파일럿에서 다음 규칙을 확인했다.

- 의자·탁자·벽처럼 지지물에 몸을 맡긴 자세는 지지물을 없애면 몸의 기울기와 접점이 무너질 수 있다. 중립 지지물 표현 계약을 별도로 정하기 전에는 현재 마네킹 포즈 경로에서 제외하거나 사람 검토 대상으로 둔다.
- 주머니에 넣은 손은 팔 높이와 방향을 보존할 수 있지만, 옷이 없는 마네킹에서는 손이 골반 실루엣과 합쳐질 수 있다. 원본에서도 손이 가려진 경우 사용할 수 있으나 검토 대상으로 기록한다.
- 빈 배경에 원 인물의 발 그림자나 접촉 그림자가 남으면 포즈 위치가 새는 것이므로 실패다.
- 마네킹 관절 조립선, 바닥 그림자, 크로마키 초록 테두리와 사람/의류 잔여물은 실패다.

## 2026-07-19 파일럿 결과와 현재 상태

- 안정 ID의 서비스용 `all`: 207/207 로컬 완료(착용 185, 제품 22).
- `pose`·`bg`: 같은 착용 예시 19개씩 로컬 완료. 나머지 착용 예시 166개는 아직 시작하지 않았다.
- 파일럿 정적 검사: 19/19 통과. 공급자 비전 검사는 호출하지 않아 QC manifest의 공식 `publishable` 판정은 아직 아니다.
- 사람 시각 검사: 포즈 18개 통과 + 주머니 손 1개 사용 가능 검토, 배경 19개 통과, 실패 0개.
- 서버 범위 계약 검사: 4/4 통과. 전용 자산 누락 시 `all`로 폴백하지 않는 규칙을 포함한다.
- 이번 파일럿의 **새 작업**에는 Gemini API를 호출하지 않았다. 다만 207개 `all`에는 앞서 만든 Gemini 3 Pro Image 156개와 Codex 내장 51개가 함께 있고 파일럿도 기존 자산을 재사용하므로, 파일럿 전체가 Codex로 만들어졌다고 표현하지 않는다. Codex 내장 생성기는 모델명이 노출되지 않으므로 Imagen 2 등 특정 모델명으로 기록하지 않는다.
- 로컬 산출물은 R2에 업로드하지 않았고, `server/app/data/example_assets.json`과 프론트 생성예시 카탈로그에도 연결하지 않았다. 현재 서버 레지스트리는 더미 3개, 프론트는 공통 placeholder 6칸 상태다.

### 2026-07-20 운영 릴리스 후속 상태

- 해시가 묶인 최종 QC에서 승인된 예시 192개만 `2026-07-19-pilot-qc-01`로 발행했다. 원본은 `all` 192·`pose` 12·`bg` 14, 파생 thumb 192개로 R2 객체 410개다.
- 보류·탈락한 `all` 15개와 미승인 보조 variant는 발행하지 않았다. R2 키·파일 크기 전수 대조 후 서버 레지스트리 v2와 프론트 카탈로그를 같은 릴리스로 교체했다.
- 제품 예시의 `gender=null`은 성별 공용으로 소비한다. 미발행 pose/bg는 계속 all로 폴백하지 않으며 UI에서 비활성화한다.
- `bg`는 10회 실생성에서 장소 반영 성공률이 약 40%에 그쳐 운영 기본값을 다시 닫았다. 프론트 `VITE_GENEXAMPLE_BG_ENABLED=true`와 서버 `GENEXAMPLE_BG_ENABLED=true`를 함께 설정한 명시적 파일럿 환경에서만 노출·저장·생성을 허용한다. QC/재시도 게이트가 마련되기 전에는 production에서 켜지 않는다.
- 아래 2026-07-19 문단의 “미업로드·미연결”은 파일럿 완료 당시의 역사적 상태이며, 현재 상태는 이 후속 기록이 대체한다.

로컬 증거는 다음 경로에 있다. `reference/genexamples/**`는 `.gitignore` 대상이므로 정본 결정은 이 ADR과 PRD·공통 계약·TODO에서 읽는다.

- `reference/genexamples/service_examples/v1/pilot_10pct.json`

## 다른 세션과의 작업 경계

현재 이미지 생성·재생성·시각 QC와 남은 166개 확장은 이 작업을 이어 온 Codex 세션이 주로 담당한다. 다른 Claude 세션의 `codex:review`는 이 ADR을 바탕으로 설계·계약·코드·테스트의 충돌을 독립 검토하는 역할이다.

별도 지시나 명시적인 인계가 없으면 다른 세션은 다음 작업을 하지 않는다.

- `reference/genexamples/service_examples/**`의 이미지·receipt·pilot JSON 생성, 교체, 이동 또는 정리
- Gemini 등 외부 유료 이미지 호출
- 남은 166개 `pose`·`bg` 생성 확대
- R2 업로드, 운영 레지스트리 교체 또는 프론트 카탈로그 배선

검토 중 생성 규칙 변경이 필요하면 코드나 자산을 임의로 바꾸기보다 근거·영향·권고안을 남겨 현재 이미지 생성 세션으로 넘긴다. 일반 코드 변경을 함께 진행할 때도 Codex의 지적은 각각 사실과 계약에 대조하고, 맞는 지적만 반영한 뒤 수정본을 다시 검토한다.

## 기존 결정과의 관계

- ADR-0004의 `all`·`bg` 의미와 예시 의류 비전이 규칙은 유지한다. 다만 `포즈만 = 포즈·프레이밍`이라는 문구 중 프레이밍은 이 ADR이 대체한다. 최신 규칙은 `포즈만 = 자세만`, 프레이밍은 현재 `shot`이 결정한다.
- ADR-0005의 사용자 섹션·내부 사진 목적·내부 `cutType` 분리는 유지한다.
- ADR-0006의 `sourceClothingType`·`applicableClothingTypes` 분리는 유지한다.
- ADR-0007의 서비스 착용 샷 `full | medium`을 프레이밍 정본으로 사용한다.
- ADR-0008에 따라 제품컷은 `ghost | detail`이며 참고 범위는 `all`만 사용한다.

## 영향

- 남은 166개를 만들기 전에 지지물 의존 포즈를 자동 후보에서 제외하거나 별도 검토 목록으로 분리한다.
- 실제 서비스 배선은 단일 manifest에서 프론트 카탈로그와 서버 레지스트리를 함께 만들고, 안정 ID·적용 의류 종류·성별·내부 생성 방식·샷·순위·실제 발행 variant를 공급하는 별도 release 작업이다.
- UI는 현재 상품과 조건에 맞는 생성예시를 최대 6장 보여주며, `pose`·`bg`는 실제 전용 자산이 발행된 항목에서만 활성화한다.
