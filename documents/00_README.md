# Wearless — 개발 핸드오프 패키지

> 패션 셀러용 AI 상세페이지 제작 스튜디오. 이 폴더는 **프론트엔드 프로토타입 → 실제 서비스(Next.js + 백엔드)** 로 이관하기 위한 인수인계 묶음이다.
> 최종 갱신: 2026-06-09

---

## 0. 이 패키지가 무엇인가

확정된 프론트엔드 프로토타입(`Wearless.html`, 빌드 없는 단일 HTML + CDN React)을 기준으로, 실제 구현팀이 바로 착수할 수 있도록 **요구사항 · 데이터/ API 계약 · 디자인 토큰 · 확정 화면 · 기술 결정**을 한 곳에 모았다.

핵심 전제:
- **디자인·인터랙션은 확정됨.** 구현팀은 "무엇을·어떻게 보이게"를 다시 고민할 필요 없이 **아키텍처·구현·백엔드**에 집중한다.
- 프로토타입의 캔버스 드래그/리사이즈/레이어 등 손으로 짠 로직은 **표준 라이브러리로 교체**될 수 있다(§ 기술 결정서).

## 1. 폴더 구성

```
handoff/
├─ 00_README.md            ← (이 문서) 인덱스 · 사용법 · 이관 순서
├─ 01_PRD.md               ← 정본 PRD (17개 섹션, 화면·정책·데이터·MVP)
├─ 02_PRD_빠른참조.md       ← 빠른 참조 (플로우·화면 핵심만)
├─ 03_기술스택_결정서.md     ← 권장 스택·라이브러리·디렉토리·이관 순서
├─ contracts/
│  ├─ types.js             ← 데이터 모델 (JSDoc 계약 + enum). 백엔드 스키마 초안
│  ├─ api.js               ← API 경계 (함수 시그니처 = 엔드포인트 매핑 대상)
│  └─ db.example.js        ← 예시 데이터 (카탈로그 + 목업). shape 참고용
├─ design/
│  └─ colors_and_type.css  ← Wearless 디자인 토큰 (color·type·spacing·radius·shadow)
└─ screens/
   ├─ 01_input.png         ← 제품 정보 입력 + AI 분석
   ├─ 02_analysis.png      ← 분석 결과 폼
   ├─ 03_mannequin.png     ← 마네킹컷 A/B + 조정
   ├─ 04_storyboard.png    ← 콘티보드
   ├─ 05_editor.png        ← 에디터
   └─ 06_library.png       ← 보관함
```

> 원본 프로토타입 코드는 상위 프로젝트(`Wearless.html` + `features/`·`components/`·`styles/`·`mock/`)에 있다. 인터랙션의 정확한 동작이 궁금하면 그 코드를 참조한다.

## 2. 읽는 순서 (역할별)

- **PM/기획** → `01_PRD.md` (정본). 미확정 의사결정(§12 크레딧, 각종 상한)을 확정한다.
- **프론트엔드** → `03_기술스택_결정서.md` → `01_PRD.md §10`(에디터) → `screens/` → `contracts/`. 화면을 쉬운 것부터 이식.
- **백엔드** → `contracts/types.js`(스키마) → `contracts/api.js`(엔드포인트) → `01_PRD.md §13~§15`(데이터·검증·안전).
- **디자이너** → `design/colors_and_type.css` + `screens/` + `01_PRD.md §14`.

## 3. 이관 순서 (요약)

1. **미확정 의사결정 확정** — 크레딧 단가, 강조특징/색상/매칭 상한, 다운로드 정책 (PRD §12, §16).
2. **기술 스택 결정 반영** — `03_기술스택_결정서.md` 채택.
3. **프로젝트 골격** — Next.js(App Router)+TS, 라우팅, 디자인 토큰 이식, 상태관리·데이터 패칭 셋업.
4. **화면 이식(정적→동적)** — 보관함 → 입력 → 분석 → 마네킹 → 콘티 → **에디터(마지막, 가장 복잡)**.
5. **백엔드 계약 구현** — `contracts/api.js`의 함수를 실제 엔드포인트로 1:1 매핑, DB 스키마(`types.js`) 구축.
6. **AI 파이프라인 · 크레딧 트랜잭션 · 인증 · 이미지 저장/내보내기**.

## 5. 알아둘 점

- 프로토타입은 **목업 데이터**로 동작한다(`contracts/db.example.js`). 실제 값·이미지는 백엔드 연동 시 대체.
- `contracts/api.js`의 모든 함수는 **async Promise** 이고, 장시간 작업은 `onProgress` 콜백을 받는다 → 실제 구현은 SSE/WebSocket/폴링 중 택1.
- **실측은 AI가 추정하지 않는다**(빈칸, 사용자 입력). 분석 응답에서 measurement value는 null.
- **세탁 안내는 분석이 아니라 에디터 자동 블록**에서 생성된다.
- 디자인 토큰은 `design/colors_and_type.css`의 `var(--*)`만 사용(57개). 임의 색·토큰 금지.
