# 마네킹 핏 선언 축 — 시장 검증 리서치

> 목적: Wearless의 마네킹컷 단계에서 **셀러가 자기 옷의 핏을 선언 → 마네킹 재현도↑**. 그 "핏 선언 축과 단계"를 감이 아니라 **관찰 근거**로 확정한다.
> 방법: 워크플로 `wf_a1728f13-4a6` (2026-07-05). 에이전트 17개 — 몰 8곳 필터/상품페이지/등록폼 직접 조사(무신사·29CM는 필터 API 직접 파싱) + 카테고리 5종 표본(상품페이지 ~150건 관찰) + 종합 + **3렌즈 회의적 검증(증거·목적·셀러UX)**.
> 원칙: 업로드 사진이 1순위. **사진에 이미 보이는 속성(색·패턴·소매길이·바지 다리라인·프린트·디테일)은 조절 축이 아니다.** 축은 "사진만으론 애매한 몸 위 핏"만 다룬다. + **생성 모델이 시각적으로 구분해 그리는가**가 최종 관문.

## 1. 핵심 발견 (관찰된 시장 구조)

1. **'전체 여유감(핏)' 사다리는 전 카테고리 보편** — 슬림→레귤러→루즈→오버 스펙트럼으로 수렴. 특히 **무신사 상품페이지 '핏' 표(스키니/슬림/레귤러/루즈/오버 사이즈 5단)는 판매자가 1개를 선언하는 구조화 필드**로, 촉감·신축성·비침·두께와 함께 노출 — **우리 목적(셀러 핏 선언)에 가장 직접적인 선례**. 단계 수는 몰별 3단(네이버·29CM·에이블리)~5단(무신사·유니클로·자라), 최빈 3~4단.
2. **'기장' 축도 전 카테고리 존재하되 라벨이 상/하의로 분화** — 상의·아우터=크롭/(숏)/기본/롱, 팬츠=7·9부/롱(+신체기준 발목), 스커트·원피스=미니/미디/롱/맥시. 몰들이 **cm보다 '몸 어디에 떨어지나'로 서술**(29CM "미니(허벅지위)/미디(무릎길이)/롱(종아리아래)").
3. **'실루엣(형태)' 축은 스커트·원피스에만 전용으로 존재** — A라인/H라인/플레어/머메이드/플리츠 등. **상의에는 별도 실루엣 축이 사실상 없다**(자라만 BOXY 1개). 팬츠 실루엣=다리라인(와이드/스트레이트/테이퍼드)은 있으나 **사진에 보이는 속성**.
4. **몰에서 가장 큰 필터(팬츠 다리라인·스커트 디테일)는 전부 '사진 가시'라 우리 축에서 제외** — 이 결론이 우리 서비스 목적과 정확히 부합.
5. **'밑위(라이즈)'는 팬츠 전용 선언 축으로 5개 몰 존재**(로우/미드/하이) — 플랫레이로 판단 어려운 대표적 사진-비가시 축.

### 2.5 성별 — 축(슬라이더 종류)은 공유, **단계 세트는 (카테고리×성별)로 분화** ← 사용자 교정 반영

**축 체계(핏·기장·밑위·실루엣)는 성별 공유.** 시장 증거: **자라 남성** 셔츠/티셔츠=슬림/레귤러/릴렉스/루즈/박시/오버(여성과 동일 사다리), **29CM 남성** 하의·아우터="여성과 동일 구조"(데이터 명시), **유니클로 MEN** 진=여성과 동일, **무신사** 핏 5단=유니섹스 필드. → 별도 남성 taxonomy는 불필요.

**단, 단계 범위는 성별로 체계적으로 다르다 (초기 "소수 전용 값" 판정은 과소평가 — 사용자 교정 2026-07-05).** 패턴: **여성은 타이트·짧은 방향으로 범위가 확장**되고, 남성은 고유값(머슬핏)을 가짐:
- 여성 상의 핏 = **타이트**/슬림/레귤러/루즈/오버 5단 — 무신사 셀러선언 5단 원본이 스키니(=타이트) 포함이었고, 에이블리(여성몰) 필터에도 타이트핏 존재. 초안이 4단으로 줄이며 여성 쪽을 깎았던 것.
- 여성 상의 기장 = **울트라크롭**/크롭/기본/롱 4단 — 무신사 기장 필터 원본(울트라크롭/크롭/하프/롱). 배꼽 노출 기장은 여성복에서 별개 상품군.
- 남성 상의 핏 = 슬림/레귤러/루즈/오버 4단 — 머슬핏은 검토 결과 **슬림에 흡수**(셀러 결정 2026-07-06, 생성에서도 남성 베이스 근육 탓에 슬림과 차이 작음).
- 팬츠 **컷**은 성별 세트 분화: (여) 스키니/슬림/일자/부츠컷/와이드 · (남) 슬림/일자/테이퍼드/와이드(배기). 아우터 단계는 남녀 동일. 스커트·원피스는 남성 게이팅(숨김).
- '여리여리한 핏' = 타이트~슬림 단계 + 소재 드레이프(소재 블록 담당)의 조합 — 별도 축 아님.

성별 처리 구조 (구현 관점):
1. **베이스 마네킹** — 이미 성별 분리(`targetGenders`→남/여 고정 베이스). 같은 선언도 남/여 몸에서 다르게 렌더.
2. **단계 세트 선택 키 = 카테고리×성별** — 카테고리 게이팅과 같은 메커니즘, 키만 확장.
3. **카테고리 게이팅** — 남성 선택 시 스커트·원피스 축 숨김.

**생성 검증 (2026-07-05, 전부 통과)**:
- 남성 베이스: 핏 사다리(슬림/레귤러/오버)·밑위(로우/하이) → 여성과 동일하게 뚜렷 (남성 체형·어깨 반영).
- 성별 전용 단계: **여성 타이트**(슬림과 구분되게 바디콘 밀착) · **여성 울트라크롭**(배꼽 위 컷, 미드리프 노출) · **남성 머슬핏**(가슴·이두 밀착) → 3개 전부 뚜렷.

## 4. 제외한 축 + 이유 (사진 가시 or 증거·구분력 부족)

- **소매기장·넥라인·칼라** (상의/원피스) — 사진 가시. (29CM·네이버 필터엔 있으나 조절축 아님)
- ~~**팬츠 다리라인** — 사진 가시라 제외~~ → **판정 번복(2026-07-06, 사용자 교정): '컷' 축으로 채택.** 근거: ①자체 테스트에서 생성이 사진의 다리라인을 안정적으로 보존하지 못함('슬림' 지시가 일자를 슬림으로 바꾼 사례) → 컷 선언은 중복이 아니라 **사진을 고정하는 닻**(소재 블록과 동일 논리: 사진에 보여도 선언하면 재현도↑) ②셀러의 바지 사고 언어 자체가 컷(전 몰 최대 필터, 어휘 빈도 1·2위) ③생성 검증: 스키니/부츠컷/와이드/테이퍼드 전부 뚜렷. 분석 단계가 사진에서 자동 추정해 채워주고 셀러는 확인/수정만.
- **스커트 디테일**(플리츠/랩/티어드/러플/셔링) — 표본 최다(플리츠 13회)지만 **원단 구조=사진 가시**. 실루엣(몸 위 퍼짐)과 분리.
- **밴딩·스트링·트임·플리츠(원턱/투턱)** — 사진 가시 디테일 + 단계형 축 아님.
- **스커트 허리선** — 몰 1곳(지그재그)뿐 = 채택 기준(2몰) 미달, 기장 축이 밑단 끝점을 이미 선언해 중복 → **3렌즈 모두 제거 판정**.
- **드롭숄더·래글런·볼륨패딩** — "사진 비가시"로 관찰됐으나 몰 필터 0곳·표본 저빈도 → **보류(추후 카테고리 전용 옵션 후보)**. 단 **패딩 부풀기**는 목적렌즈가 "교과서적 사진-비가시 핏"이라 재검토 권고.
- **신축성·두께·비침** — 핏 재현 유관하나 **소재 파이프라인**(이미 구현한 소재 블록) 영역.

## 5. 남은 검증 + 결정

## 7. 독립 리뷰 반영 (Codex 22건 + 자체 7건, 2026-07-06)

### 7.1 닻(anchor) 규칙 명문화 — "사진에 보이면 제외"의 예외 기준 (Codex #1, HIGH)
팬츠 컷 번복으로 원칙 경계가 모호해진 것을 교정. **닻 채택 3조건(모두 충족 시에만)**:
1. **셀러-중요 속성**이고 (컷·소재처럼 상품의 정체성)
2. **생성이 자주 드리프트**하며 (실측 관찰로 입증 — 컷·소재가 해당)
3. **단계형 축으로 표현 가능**할 것 (소매·넥라인·프린트·디테일은 ③ 불충족 → 레퍼런스 이미지 몫)
→ 현재 닻 = 소재 블록 + 팬츠 컷 2개뿐. 후보(패딩 볼륨 등)는 드리프트 입증 후에만 추가.

### 7.2 구현 스펙 필수 항목 (통합 갭 — 스펙 문서에 반드시 포함)
- **`fitProfile` 계약 신설**: `{ category(top|pants|skirt|dress|outer — clothingType+subCategory에서 유도), gender, axes{...}, source(auto|seller), confidence, version }` — Analysis·MannequinCut·garment_ref에 배선 + 마이그레이션. (기존 `Fit`/`AdjustFit`/`matchAdjust`/`candidate` 필드 대체)
- **백엔드**: `candidate_specs()`+`_FIT_CONTRAST` 폐기 → 프로필 기반 단일 spec. 프롬프트 `${baseFit}` → 카테고리별 **FIT PROFILE 블록** + **우선순위 재작성**(선언 프로필 > 핏 참고사진 > 기본 추정 — 현행 "핏 뷰 우선" 문구와 충돌 해소).
- **프론트**: Mannequin.jsx의 A/B 구조(cutsA/B·히스토리·문구) 철거 → 단일컷+히스토리+재생성 루프. **분석 페이지 기존 '핏' 필드 = 프로필의 초기 자동추정값으로 통합**(중복 노출 금지). 매칭 의류는 우리 시드 메타로 **자동 적용**(matchAdjust 제거).
- **크레딧 매트릭스**: 프로필 선언·수정=무료 / 재생성=과금(비용 미리보기+이전 결과와 비교 UI) / 2회 제한 폐지.
- **자동 추정 UX**: 추정값에 "AI 추정" 뱃지, 낮은 확신이면 명시적 확인 요구 (조용히 틀린 채 전파 방지).

### 7.3 백로그 (설계 결함 아님, 후속)
- 모순 조합 경고(반팔+롱 등), 디테일 손실 전용 재생성 사유("디테일이 달라요"), measurements(실측 cm)와 시각 기장 선언의 우선순위 정의, 패딩 볼륨 닻 후보 검증, 아우터·원피스 타일 에셋 제작(자체 생성으로 조달 — 비용 한정적).
