# wearless.kr 랜딩페이지 설계

> 상태: 설계 확정 (2026-07-05)
> 대상: wearless.kr — 별도 Next.js 프로젝트로 구현 (이 저장소의 Vite SPA와 무관)
> 서비스 앱: ai.wearless.kr (현재 저장소)
> 목적: 검색·광고 유입을 받아 서비스로 전환시키는 마케팅 랜딩. SEO 기반 확보.

---

## 1. 전제와 결정 사항

| 결정 | 내용 |
|---|---|
| CTA 전략 | 메인 CTA는 **"무료로 시작하기" → ai.wearless.kr 직행**. 사전 등록 폼 없음 |
| 가격 섹션 | **임시 플랜 표 공개**. 단, "출시 예정가" 라벨 + "정식 출시 시 변경될 수 있습니다" 고지 필수 |
| 결과물 예시 | 섹션 제작 + **노출**. 스파이크 실생성물 중 베스트 2~3개 사용, 데이터 교체 가능 구조 |
| 후기 섹션 | 컴포넌트·데이터 구조만 준비, **데이터 없으면 렌더링 스킵** (v1 비노출) |
| 스코프 | **원페이지** + 법적 페이지(이용약관, 개인정보처리방침) |
| 스택 | Next.js App Router, 정적 렌더링 우선, `next/image` |

원칙: "빈 증명 섹션을 공란으로 노출하지 않는다." 데이터가 없는 섹션은 코드에는 존재하되 화면에서 빠진다. 데이터가 생기면 콘텐츠 파일만 채워서 켠다.

## 2. 섹션 구성 (위→아래 순서)

### 2.1 Hero — 지금 작성
- H1 한 줄 가치 제안. 방향: "상품 사진 몇 장으로, 상세페이지 한 장을." (카피는 구현 시 확정)
- 서브카피: 모델·촬영·디자인 없이 AI가 착용컷과 카피까지 만든다는 요지.
- 메인 CTA: `무료로 시작하기` → https://ai.wearless.kr
- 보조 CTA: `결과물 보기` → 결과물 예시 섹션 앵커 스크롤.
- 비주얼: 생성된 상세페이지 세로 스크롤 목업 또는 베스트 생성컷. 결과물 예시와 동일한 콘텐츠 소스를 공유.

### 2.2 문제 공감 — 지금 작성
1인·소규모 패션 셀러의 페인포인트 3개 내외: 모델 섭외·촬영 비용, 상세페이지 외주 비용과 리드타임, 컬러별·컷별 이미지 준비 부담. (PRD §2 기반)

### 2.3 How it works — 지금 작성
서비스 4단계 스텝퍼를 그대로 보여준다 (PRD §3):
```
제품 정보·분석 → 마네킹컷 → 콘티보드 → 에디터
```
단계별 한 줄 설명 + 해당 화면 스크린샷(또는 단순 일러스트). 스크린샷은 현재 앱 화면으로 충분하며 추후 교체.

### 2.4 기능 하이라이트 — 지금 작성
카드 4~6개: 컷 종류(스타일링·호리존·제품컷·거울샷), 컬러별 이미지 생성, 마네킹 기반 핏 기준, 자동 카피라이팅, 캔버스 에디터, 크레딧 사전 고지. (PRD §7~§12 기반)

### 2.5 결과물 예시 (Before/After) — 노출
- 구조: 셀러 원본 사진(before) ↔ 생성 컷/상세페이지(after) 비교. 2~3세트.
- 콘텐츠 소스: 스파이크 실생성물 중 품질 베스트를 선별해 사용. 정식 결과물이 쌓이면 교체.
- AI 생성 이미지임을 명시하는 캡션 포함 (PRD §15.4 고지 원칙과 일치).

### 2.6 후기/사례 — v1 비노출
- 컴포넌트와 데이터 스키마(이름, 쇼핑몰명, 인용문, 아바타)만 만들어두고, 데이터 배열이 비어 있으면 섹션 자체를 렌더링하지 않는다.

### 2.7 가격 — 임시 플랜 표
- 구독 플랜 + 크레딧 추가 구매 구조를 반영한 표 (크레딧 시스템 설계와 정합: 구독제 + 추가 구매, 월 리셋).
- 표 상단에 `출시 예정가` 배지, 하단에 "정식 출시 시 구성과 가격이 변경될 수 있습니다" 고지 — **이 두 요소 없이 배포 금지**.
- 숫자는 콘텐츠 파일에서 관리해 확정 시 즉시 교체.

### 2.8 FAQ — 지금 작성, SEO 핵심
5~8문항. 후보: "AI 생성 이미지를 상세페이지에 써도 되나요?", "생성 이미지가 실제 상품과 다르면?", "크레딧은 언제 차감되나요?", "어떤 의류를 지원하나요?", "결과물 수정이 가능한가요?", "다운로드 형식은?".
`FAQPage` JSON-LD 구조화 데이터 필수.

### 2.9 최종 CTA — 지금 작성
Hero와 동일한 CTA 반복 + 짧은 마무리 카피.

### 2.10 Footer + 법적 페이지 — 지금 작성
- 사업자 정보(상호, 대표, 사업자등록번호, 연락처 — 전자상거래 표시 의무 대비), 이용약관, 개인정보처리방침 링크.
- `/terms`, `/privacy` 별도 라우트.

## 3. 콘텐츠 아키텍처

섹션 콘텐츠(결과물 예시, 후기, 가격, FAQ)는 컴포넌트에 하드코딩하지 않고 콘텐츠 데이터 파일(예: `content/*.ts`)로 분리한다.
- 배열이 비면 해당 섹션 비노출 — "공란" 처리의 구현 방식.
- 서비스 완성도에 따라 코드 수정 없이 데이터만 갱신.

## 4. SEO 체크리스트

- [ ] `metadata` (title, description) — 타깃 키워드: "AI 상세페이지", "상세페이지 제작", "쇼핑몰 상세페이지", "의류 상세페이지"
- [ ] OG 이미지 + 트위터 카드
- [ ] H1은 페이지당 1개, 키워드 포함
- [ ] `FAQPage` JSON-LD
- [ ] 시맨틱 마크업 (`section`, `nav`, `footer`), 모든 이미지 alt
- [ ] `sitemap.xml`, `robots.txt`
- [ ] wearless.kr ↔ ai.wearless.kr 상호 링크 (서비스 앱 푸터/로고에서 랜딩 역링크)
- [ ] 정적 렌더링 (전 페이지 SSG 가능 — 동적 데이터 없음)

블로그/가이드 콘텐츠는 v1 범위 밖. 검색 유입 확장이 필요해지는 시점에 별도 결정.

## 5. v1 범위 밖 (명시적 제외)

- 사전 등록/이메일 수집 폼
- 후기 섹션 노출 (데이터 확보 후)
- 블로그, 기능 상세 개별 페이지
- 다국어(영문)
- A/B 테스트 인프라
